qrwar

qrwar

  • Offizielle Webseite
  • Code Repository

qrwar (quad-ruled war) ist ein Pen & Paper ähnliches Rundentaktikspiel, bei dem Schlachten auf einem in Quadrate unterteilten Spielfeld stattfinden. Es handelt sich um ein kleines Projekt um 2D-Programmierung, vor allem mit SFML 2, zu erlernen.

Gameplay

In qrwar treten sich zwei Generäle verfeindeter Armeen auf riesigen Schlachtfeldern, die nach belieben gestaltet werden können, gegenüber. Das Gameplay lehnt sich an spiele wie Advanced Wars von Nintendo an, vertritt jedoch einen simpleren Ansatz: Das Spiel soll auch auf einem Stück kariertem Papier spielbar sein, ohne dabei komplexe Regeln oder Kampfsysteme zu verwenden.

Das Spiel läuft in mehreren Schritten ab:

  1. Zusammenstellen der Armee
  2. Platzieren von Gelände und Aufstellen der Soldaten
  3. Die Schlacht

Features

  • Rundenbasierte Taktik
  • 2-Spieler hot seat: Tritt gegen einen Freund (oder Feind) an einem PC an
  • Passe die Landschaft der Schlachtfelder nach deinen Wünschen an
  • Einfachste Spielmechaniken
  • Zukünftig weitere Features!

Get it!

Hinweis: qrwar befindet sich noch in der Entwicklung, weshalb noch keine Releases vorliegen. Wenn die erste Version (v0.1) fertig gestellt wurde, werden vorcompilierte Binärpakete zur Verfügung gestellt. Bis dahin muss qrwar eigenhändig compiliert werden.

Quellcode

Der Quellcode ist auf github verfügbar. Dort findet sich auch eine Compilier-Anleitung (die im Moment eventuell noch nicht fertig ist. Ich bitte dies zu entschuldigne :) ).

Technologie

qrwar wurde in C++ unter Zuhilfenahme der 2D-Grafik Bibliothek SFML geschrieben. Für die GUI kommt SFGUI zum Einsatz, die jedoch teilweise durch eine "Bibliothek" names nameless-gui, die ich speziell für meine Ansprüche entwickelt habe, ersetzt wurde.
